# 桌面宠物打包指南

## 准备工作

1. 确保已安装Python和所有依赖
2. 安装PyInstaller：
   ```
   pip install pyinstaller
   ```

## 打包步骤

### 方法一：使用批处理脚本（推荐）

1. 双击运行 `build.bat`
2. 等待打包完成（可能需要几分钟）
3. 打包完成后会自动打开输出目录

### 方法二：手动打包

在项目根目录运行：
```bash
pyinstaller build_pet.spec --clean
xcopy /E /I /Y assets dist\桌面宠物\assets
```

## 打包结果

打包完成后，`dist\桌面宠物\` 文件夹包含：

```
桌面宠物\
├── 桌面宠物.exe          # 主程序
├── assets\                # 美术素材文件夹（用户可替换）
│   ├── idle.gif
│   ├── walk.gif
│   ├── fall.gif
│   ├── ...
│   └── icon.ico
├── _internal\             # 运行时依赖（不要动）
│   ├── Python DLL
│   ├── PyQt6
│   └── ...
└── 使用说明.txt           # 用户使用说明
```

## 发布准备

1. **压缩整个文件夹**：
   - 右键 `桌面宠物` 文件夹
   - 发送到 → 压缩(zipped)文件夹
   - 得到 `桌面宠物.zip`

2. **告诉用户**：
   - 解压后运行 `桌面宠物.exe`
   - 更换外观：替换 `assets` 文件夹里的GIF文件
   - 配置文件在：`C:\Users\用户名\.desktop_pet`

## 用户如何更换美术素材

1. 进入 `assets` 文件夹
2. 准备同名的GIF文件（建议512x512）
3. 替换对应文件：
   - `idle.gif` - 待机动画
   - `walk.gif` - 行走动画
   - `fall.gif` - 掉落动画
   - `drag.gif` - 被拖拽
   - `sleep.gif` - 睡觉
   - 等等...
4. 重启桌宠即可

## 常见问题

### Q: 打包后exe体积很大？
A: 正常现象，因为包含了Python运行时和PyQt6。使用UPX可以压缩（已在.spec中启用）。

### Q: 用户说找不到某个DLL？
A: 确保 `_internal` 文件夹和exe在同一目录。

### Q: 用户替换GIF后没效果？
A: 检查文件名是否完全一致（包括大小写），重启桌宠。

### Q: 想打包成单文件exe？
A: 不推荐，因为用户无法替换美术素材。如果一定要：
```python
# 在build_pet.spec中修改
exe = EXE(
    ...
    exclude_binaries=False,  # 改为False
    ...
)
# 然后删除COLLECT部分
```

## 测试建议

打包后务必测试：
1. ✓ 双击exe能否正常启动
2. ✓ 替换assets里的GIF，重启后是否生效
3. ✓ 右键菜单、设置窗口是否正常
4. ✓ AI功能是否正常（需要配置API Key）
5. ✓ 在没有Python环境的电脑上测试
